css transform2D vs transform3D变换

硬件加速的极限

当浏览器决定一个元素应当使用硬件来加速时，它不能把原生的DOM元素传给GPU。相反，它将元素呈现到一个图片中，并向GPU发送图像。GPU将图像应用为多边形纹理贴图来代表元素。然后GPU可以非常迅速地移动这些多边形。如果位图有透明信息（Alpha通道），该元素就能部分透明。GPU还可以缩放元素、旋转或原声地应用变换。它不能重新渲染内容。一旦元素被发送到GPU，它只能是一个图像内容，而不是实际的内容。

比较2D变换与3D变换


由于2D变换不需要渲染图像，浏览器能够以较大的尺寸重新渲染文本。CPU只有图像可供操作，所以它只能按比例放大图像，从而得到一个模糊的版本。
要解决这个限制，一旦动画完成，我们可以“净化”变换的元素。除非我真的要做一些3D效果，否则可以在动画完成时，通过在触发
transitionend或animationend事件时改变样式，将3D变换转换回2D变换。

净化变换还有一个好处：释放GPU占用的内存。
如果你应用了多个变换，最好在DOM外保持元素的状态，这样你无需再次从DOM树中读取元素。
（意为在JavaScript变量中保存该DOM或其相关属性）
